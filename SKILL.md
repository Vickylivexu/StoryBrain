# StoryBrain - 故事化知识学习系统

StoryBrain 是一个结合 **concept-fable（概念寓言）** 和 **hv-analysis（纵横分析）** 的知识学习系统。它通过五个核心模块帮助用户深度理解任何概念，输出可直接导入 Obsidian 等笔记工具的知识库。

## 核心方法论

### Concept Fable（概念寓言）

用隐喻故事解释抽象概念。**完整结构**：

| 字段 | 说明 | 要求 |
|------|------|------|
| `title` | 故事标题 | 如「藏书阁的秩序与岁月」 |
| `story` | 完整故事文本 | ≥ 200 字，生动有画面感 |
| `explanation.conceptName` | 概念名 | 如「熵增定律」 |
| `explanation.oneSentence` | 一句话定义 | 30 字以内 |
| `explanation.whyItMatters` | 为什么重要 | 50-100 字 |
| `explanation.keyPoints` | 核心要点 | 3-5 条 |
| `explanation.mapping` | 故事元素→概念映射 | ≥ 3 条，精确对应 |
| `explanation.foodForThought` | 值得思考的问题 | 开放式 |
| `explanation.relatedConcept` | 相关概念延伸 | 含故事续写 + 概念介绍 |

### HV-Analysis（纵横分析）

**纵轴（Vertical）**：概念的起源、演变脉络、关键里程碑

**横轴（Horizontal）**：与相似/相对概念对比，在领域中的定位

**洞察与未来**：综合分析 + 发展展望

## 触发条件（INVOKE WHEN）

用户说以下内容时，调用 story-brain：

- "我想深入了解一下 XXX"
- "帮我分析一下 XXX"
- "用故事解释 XXX" / "用寓言说明 XXX"
- "我想了解 XXX" / "帮我研究 XXX"
- "横向对比" / "纵向追溯" / "系统研究"
- "把 XXX 整理成笔记" / "导出 Markdown"
- "帮我分析大学专业 XXX"

## 完整工作流程

### Step 0: 确定分类

用户输入后，判断主题类型：

- **大学专业**：用户明确提到「专业」「大学」「学科」，或输入名称在 `data/` 目录匹配到 `majorInfo` 字段
- **通用概念**：其他所有情况

### Step 1: 数据查找

优先检查 `data/` 目录是否有现成数据：

```
用户输入「区块链」
  ↓
查找 data/blockchain.json 或 data/区块链.json
  ↓
存在 → 加载已有数据 → 直接构建输出
不存在 → 进入 Step 2 实时生成
```

### Step 2: 生成概念寓言

**调用 concept-fable skill**，传入主题名，生成完整寓言故事。

**寓言生成 Prompt 指导**：

```
主题：[用户输入的概念]
类型：[大学专业 / 通用概念]

故事要求：
- 背景贴近生活（村庄、藏书阁、花园、工厂等）
- 主角清晰（园丁、馆长、工匠等）
- 情节有起伏（危机→应对→启示）
- 故事长度：200-500 字
- 映射表：每个故事元素精确对应一个概念要素
```

### Step 3: 纵横分析

**调用 hv-analysis skill**，对主题进行三章深度研究。

### Step 4: 构建知识卡片

从生成内容中提取：

- 5-8 个核心概念（含定义）
- 3-6 个延伸学习主题
- 知识图谱（节点 + 边）

### Step 5: 生成分类标签

根据主题类型生成对应标签：

**通用概念**：

| 字段 | 说明 | 示例 |
|------|------|------|
| `category` | 学科大类 | 物理学、计算机科学、经济学 |
| `subCategory` | 子领域 | 热力学、分布式系统 |
| `coreConcepts` | 核心概念清单 | [熵, 热力学第二定律, ...] |
| `applicationDomains` | 应用领域 | [物理化学, 信息论, ...] |
| `relatedFields` | 关联学科 | [统计力学, 量子信息, ...] |
| `difficulty` | 难度等级 | 入门 / 进阶 / 专业 |
| `learningPath` | 学习路径 | [步骤1, 步骤2, ...] |
| `customTags` | 自定义关键词 | [热力学, 熵, 时间箭头, ...] |

**大学专业**：

| 字段 | 说明 |
|------|------|
| `majorInfo` | 学科分类信息对象 |

```json
{
  "subjectCategory": "学科大类",
  "subCategory": "子领域",
  "degreeType": "工学/理学/管理学/文学...",
  "difficulty": "入门/中等/高/非常高",
  "keyCourses": ["课程1", "课程2", ...],
  "specializations": ["方向1", "方向2", ...],
  "careerPaths": ["就业路径1", "就业路径2", ...],
  "relatedMajors": ["相近专业1", "相近专业2", ...],
  "subjectAssessment": "学科特点与就业建议"
}
```

### Step 6: 质量验证（必做）

对输出内容运行 14 项质量检查。若发现问题，重新生成对应部分。

| # | 检查项 | 通过标准 | 不通过信号 |
|---|--------|---------|-----------|
| 1 | 主题名纯净 | 无「新概念」「待研究」等占位词 | 含占位词 |
| 2 | 寓言字数 | story ≥ 200 字 | < 200 字 |
| 3 | 寓言标题 | 非默认标题 | 「关于XXX的故事」 |
| 4 | 一句话定义 | 非占位定义 | 含「是一类...」「是一种...」开头 |
| 5 | 映射表条目 | ≥ 3 条 | < 3 条 |
| 6 | 核心概念真实性 | 5个中≥3个为真实术语 | 含「XX的核心概念」占位 |
| 7 | 概念定义充实 | 每条定义≥20字 | 过短或为空 |
| 8 | 延伸主题相关性 | 3个中≥2个与主题相关 | 含主题名本身 |
| 9 | 纵向分析字数 | ≥ 300 字 | < 300 字 |
| 10 | 横向对比字数 | ≥ 300 字 | < 300 字 |
| 11 | 洞察章节字数 | ≥ 200 字 | < 200 字 |
| 12 | 标签分类有效 | category/subCategory 非「通用知识」 | 含通用占位 |
| 13 | 无占位符文本 | 不含「待完善」「待研究」等 | 含占位符 |
| 14 | 难度等级有效 | 非「待评估」 | 含待评估占位 |

**质量等级**：

| 等级 | 评分 | 说明 |
|------|------|------|
| high | ≥ 80 分 | 可直接交付 |
| medium | 55-79 分 | 警告但可交付 |
| low | < 55 分 | 不应交付，需重写 |

### Step 7: 输出

根据用户需求，选择输出方式：

1. **直接展示**：在对话中展示完整研究内容
2. **生成 Markdown**：输出 Obsidian 兼容格式

## 使用提示

1. **优先使用预生成数据**：有现成数据时加载使用，保证质量
2. **故事要贴近生活**：选择用户熟悉的场景作为故事背景
3. **映射要精确**：每个故事元素必须清晰对应一个概念要素
4. **纵横结合**：不要孤立分析，纵向追溯发展，横向对比关联
5. **质量验证不跳过**：low 质量或含占位符的内容不应交付
6. **大学专业展示简化**：前端只展示学科大类、子领域、学习难度；Markdown 导出保持全部字段
7. **保存新数据**：对新生成的高质量研究，可保存到 `data/` 目录供后续使用
